# Назначение

Используйте этот skill, когда необходимо подготовить документацию к модулю, классу, методу или CLI-сценарию в формальном стиле.

# Обязательные правила

1. Язык описания: русский.
2. Тон: официальный, нейтральный, без разговорных оборотов.
3. Структура: фиксированная, с повторяемыми разделами.
4. Источник истины: только фактический код, тесты и наблюдаемое поведение.
5. Примеры должны опираться на реальные сигнатуры и существующие CLI-сценарии.

# Структура документа

Используйте следующий порядок, если пользователь не запросил иной формат:

1. Назначение
2. Основные сущности
3. Публичные методы или точки входа
4. Параметры и возвращаемые значения
5. Примеры использования
6. Практические замечания

# Ограничения стиля

Не используйте:

- разговорные конструкции;
- эмоциональные оценки;
- маркетинговые формулировки;
- неподтвержденные утверждения о поведении.

Предпочтительно использовать:

- короткие абзацы;
- таблицы для параметров;
- формулировки вида «Назначение», «Параметры», «Возвращаемое значение», «Пример».

# Порядок работы

1. Сначала проанализируйте исходный файл.
2. Затем выделите публичные сущности.
3. После этого постройте строгий каркас документа.
4. Только затем формируйте окончательный текст.
5. Перед завершением проверьте полноту покрытия и наличие примеров.

# Критерии качества

Документ считается готовым, если:

1. Описаны все публичные классы, функции и методы.
2. Для каждого метода указаны параметры и результат.
3. Есть не менее одного реалистичного примера использования.
4. Текст выдержан в едином официальном стиле.
