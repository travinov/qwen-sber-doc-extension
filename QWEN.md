# Gigadoc Extension (Qwen Format)

Используйте это расширение, когда требуется задокументировать Python-модуль в официальном русскоязычном стиле со строгой структурой.

Правила применения:

1. Для редакционного стиля и структуры сначала ориентируйтесь на skill `sber-doc-style`.
2. Для документирования конкретного файла используйте команду `/doc:sber <path>`.
3. Для фактического анализа исходного кода и проверки полноты вызывайте MCP tools:
   - `analyze_python_target`
   - `build_sber_doc_outline`
   - `build_sber_project_outline`
   - `validate_sber_doc`
4. Не смешивайте официальный стиль с разговорной подачей.
5. Не придумывайте параметры, поведение и примеры. Если данные не подтверждены кодом, явно помечайте ограничение.

Рекомендуемый порядок работы:

1. Вызвать `analyze_python_target` для файла или директории.
2. Если `kind=module`, построить каркас через `build_sber_doc_outline`.
3. Если `kind=project`, построить каркас через `build_sber_project_outline`.
4. Сгенерировать финальный текст, соблюдая skill.
5. Проверить итог через `validate_sber_doc`.
