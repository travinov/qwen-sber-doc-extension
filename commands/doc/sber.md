Подготовьте документацию в стиле Сбера для целевого Python-модуля `{{args}}`.

Порядок работы обязателен:

1. Вызовите MCP tool `analyze_python_module` для файла `{{args}}`.
2. На основе результата вызовите `build_sber_doc_outline`.
3. Сформируйте финальную документацию на русском языке в официальном нейтральном тоне.
4. Обязательно включите:
   - назначение модуля;
   - основные сущности;
   - параметры публичных методов;
   - возвращаемые значения;
   - примеры использования;
   - практические замечания.
5. Проверьте итог через `validate_sber_doc`.

Требования к стилю:

- без разговорных формулировок;
- без смешения русского и английского в описательных абзацах;
- без домыслов о поведении, не подтвержденных кодом.

Если входной путь не существует или файл не является Python-модулем, явно сообщите об этом и не создавайте фиктивную документацию.
